Tutorial

  1. Strict Mode, use it to produce better code

    'use strict';
    more errors info, block some features to work with less problems
    not defined variables are trown as exceptions
    reserves some words as future lang features

  2. Functions - like variable/container for chunk of code

    function logger() { ... }
    logger();
    logger();
    logger()
    Code reuse
    input, output; analogy to machine
    function logger(x,y) { const z = `${x},${y}`; return z; }
    const valz = logger(valx,valy);
    Allow more maintanable code, DRY - dont repeat yourself
    Functions can call functions.

  3. Functions Declarations VS Expressions

    Declaration is named function as shown in above section point.
    Function Expression is annonymous function that can be stored as variable.
    const calcAge = function (birthYear) { return 2037 - birthYear; }
    const age = calcAge(1991);
    Function Declarations can be called/used before it's declaration. (unlike Functions Expressions)
    Use what you preffer.

  5. Arrays

    Create: const friends = ['Michael', 'Steven', 'Peter']; (cant reassign array)
    Create: const years = new Array('1991', '1984', '2008', '2020');
    Read: years[0]; years[2]; years.length; years[years.length - 1]
    Update: friends[2] = 'Jay';
    Values, Expressions, Variables, Arrays, Functions can be used as elements of Array.
