Description

Quickstart for node.js in azure.

Prerequisites/Assumptions

  1. Azure account with an active subscription.
  2. Install Node.js and npm.
  3. Install Visual Studio Code.
  4. The Azure App Service extension for Visual Studio Code.

Tutorial

  1. Create new github repo, open git bash in Code folder, clone it.

    git clone https://github.com/atari-monk/web-dev-basics.git

  2. Verify that Node.js is installed.

    node --version

  3. Create a Node.js application using the Express Generator.

    details

    repo folder

    app name

    cd web-dev-basics && npx express-generator myExpressApp --view ejs

  4. Install the NPM packages to the app.

    details

    app name

    cd myExpressApp && npm install

  5. Start the development server with debug information.

    details

    app name

    DEBUG=myExpressApp:* npm start

  6. Test in the browser.

    http://localhost:3000

  7. Deploy to cloud using VSCode Azure extension.
    Sign in. Azure email should be in status bar and subscription in explorer.

  8. Right-click on App Services, Create new Web App. A Linux container is used by default.

  9. Name unique in Azure ('A-Z', 'a-z', '0-9', '-').

    details

    example name

    atari-monk-myExpressApp

  10. Select Node runtime stack and Node.js latest LTS (long term support) version.

  11. Select pricing tier Free (F1).

  12. All options are in right click menu of App Services in explorer.

  13. Error - Couldn't detect a version for the platform 'nodejs' in the repo.

  14. In .vscode/settings.json paste (for Azure extension to deploy without above error)

    details

    app repo name

    app proj folder

    "appService.defaultWebAppToDeploy": "web-dev-basics"
    "appService.deploySubpath": "myExpressApp"

Summary

Node js app template was done.
It was deplyed on Azure, after providing basic settings.
