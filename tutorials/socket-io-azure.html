Description

 Socket.IO enables bidirectional communication between (Node.js) server and (Browser/Node.js) client.
It's most popular on the web for this.
 Official get started combines server and client under one namespace. I tested it locally.
When i deploy on azure cloud, i prefer them separated, so i used Source tutorial.
Code in repo socket-io-qs.

Prerequisites/Assumptions

  1. Install VSCode
  2. Install Node.js

Tutorial

  1. Setup project. In git bash or VSCode terminal.

    cd chat-server
    npm init
    npm install socket.io
    npm install express

  2. To .gitignore add (if not there)

    node_modules
    .vscode

  3. Add index.js with server code.

  4. Run it locally. Ctrl+C to kill it.

    node index.js

  5. Add folder chat-client.

  6. Add index.html with chat client code.

  7. With live server extension, open two instances of chat client page.

  8. Open git bash in chat-server folder and run server.

    node index.js

  9. Send message with one client. Check if it was recive by server and send to another clinet.

  10. Deploy to Azure.

  11. VSCode Azure extension. App Services. Create New Web App.

    details

    example name

    atari-monk-chat-server

  12. Right click on chat-server, Deploy to Web App... .

  13. Choose web app name and assign folder to it in popup.

  14. After server is deployed. Deploy client page with github pages.

  15. Test it.
