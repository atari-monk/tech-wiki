Description

'Secrets' is json file per project, stored in windows AppData folder.
It stores key, value pairs with passwords and data you dont want to commit to repo.
Source tutorial contains more usefull info and commands.

Prerequisites/Assumptions

  1. Powershell
  2. VSCode

Tutorial

  1. Remove plain password from appsettings.json.

  2. Add connection string builder or config class (using Configuration secrets) to programm.cs.
    Use it in ef core db context config.

  3. Add guid key for secrets to csproj file.

    dotnet user-secrets init

  4. Add secret key value pair.

    details

    key

    value

    dotnet user-secrets set "DbPassword" "x"

  5. Show secrets list.

    dotnet user-secrets list

  6. Remove secret.

    details

    key

    dotnet user-secrets remove "DbPassword"

  7. Clear all secrets for proj.

    dotnet user-secrets clear

Summary

This is a way to not commit configs with passwords and other data to repos.
